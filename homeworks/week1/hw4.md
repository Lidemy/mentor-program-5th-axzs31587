## 跟你朋友介紹 Git

### Git 的基本概念
就像菜哥你說的那樣，Git 是用來幫我們做版本控制的程式，簡單說明 Git 的概念，假設菜哥你現在的資料夾中有版本一到版本四的文字檔、一些有讓你產生靈感的梗圖和一些時事的新聞稿來讓你的笑話跟上流行，但光是這樣看可能看不出來哪些梗圖是跟哪個版本的笑話有關，而哪些版本的笑話又有講到哪些時事，所以為了能方便知道在哪些時機點哪些檔案之間有互相依賴，我們新增四個資料夾（版本一、版本二、版本三、版本四），各資料夾的內容如下：
1. 版本一資料夾：版本一文字檔
   因為版本一的笑話沒有想很多，所以只有一個版本一的文字檔。
2. 版本二資料夾：版本二文字檔、5 號梗圖
   因為看了 5 號梗圖想到了一些這個梗的笑話進去，所以版本二的資料夾中有從版本一文字檔那邊改的－版本二文字檔和 5 號梗圖。
3. 版本三資料夾：版本三文字檔、5 號梗圖、3 號時事文字檔
   因為版本二加的那些梗意外的可以跟 3 號時事結合，所以增加了這個時事進去笑話裡，現在版本三的資料夾中就會有從版本二文字檔那邊改的－版本三文字檔、5 號梗圖和 3 號時事文字檔
4. 版本四資料夾：版本四文字檔、5 號梗圖、3 號時事文字檔、1 號梗圖、4 號時事文字檔
   版本四的笑話為了成為電視笑話冠軍，一次加入 1 號梗圖和 4 號時事進去，讓評審覺得菜哥是個有內涵又會講笑話的賣菜帥哥，所以現在版本四的資料夾就會有從版本三文字檔那邊改的－版本四文字檔、5 號梗圖、3 號時事文字檔、1 號梗圖和 4 號時事文字檔
而其他沒有用到的梗圖和時事的文字檔就不會被加入以上這些資料夾中。這樣的方法能夠清楚知道在哪些版本中有做了哪些更動，而哪些檔案之間是有互相依賴的，雖然實際上 Git 不是以資料夾來做這樣的操作，但這邊以資料夾的方式來舉例讓蔡哥更好理解。

但上面這些是菜哥一個人在做的版本控制，假如今天電視笑話的節目組要菜哥跟嘎哥一起組隊講笑話，假設菜哥和嘎哥共享了以上這些資料夾，並且都用版本四的笑話來做新笑話的延伸。
過了幾天，菜哥在電腦上寫好了叫版本五的文字檔要上傳到共用資料夾，而嘎哥也不甘示弱的在他的電腦上也寫了一個叫版本五的文字檔要上傳到共用資料夾，這時就會產生一個問題－叫版本五的資料夾中不知道要放哪個內容的版本五文字檔，所以我們不能使用這樣流水號的方式（版本一、版本二、版本三、......）來為資料夾取名，改為使用一種看起來像亂數的東西，這種像亂數的版本號絕對不會重複，所以我們用這樣的方式就能保證大家的資料夾名稱都不會重複，但這樣又產生兩個問題，第一個是我們會不知道這些資料夾的順序應該怎麼放，所以需要另外開一個檔案來記錄這些版本號的順序是什麼；而第二個問題是沒辦法知道最新的版本是哪個，所以再開一個新的檔案，裡面存最新的版本號。

而以上所有事情大概就是 Git 會幫我們做的事，以資料夾的概念來理解的話就是：
1. 需要新版本：開一個資料夾
2. 不想加入版本控制：不要加入資料夾
3. 避免版本號衝突：用看似亂數的東西當作資料夾名稱
4. 知道最新版本：用一個檔案來存最新版本的資料夾名名稱

### Git 基本指令使用流程
接著菜哥只要照以下順序輸入一些 Git 的基本指令，就能來用 Git 做版本控制囉。
但是首先，你可以到網路上搜尋 Git 然後進官網下載檔案安裝在你電腦上，安裝完後打開 Git Bash，就可以在這上面輸入以下這些 Git 的基本指令了：
1. git init：初始化
   Git 會在你目前所在資料夾中建立一個空的隱藏檔案 .git 作為初始化檔案，讓它可以存放一些版本控制需要的東西。
2. 在你目前所在資料夾中建立一個叫 .gitignore 的檔案
   因為 Git 會忽略任何放在 .gitignore 裡的檔案，不會把它們放進版本控制中，所以我們手動建立一個叫 .gitignore 的檔案，來忽略一些不要的檔案（像是電腦作業系統產生的的一些設定檔，或是我們平常會用的一些檔案等）。
3. vim .gitignore：將想要忽略的檔案寫進 .gitignore 檔案中
   vim 指令是文字編輯器的功能，因為要將不需要的檔案寫進 .gitignore 檔案中，所以使用 vim 指令，進入文字編輯器後按 i 進入 INSERT 模式才可以打字，將不要的檔案都打進去之後，按 ESC 進入普通模式，並按 :wq 即可儲存並退出文字編輯器。
4. git add . ：把目前所在資料夾中的所有檔案加進版本控制
   git add 就是決定是否加入版本控制，而 . 代表所有檔案，可以想像成要把所有檔案移進去這個版本的資料夾中。
5. git commit -am "commit messege"：建立第一個 commit
   git commit 就是新建一個版本，後面要加入 commit messege 來簡單說明一下這個版本的內容，可以想像成成功建立第一個版本的資料夾並為這個資料夾加一些說明。
6. git push origin master：將本地電腦的檔案放到遠端 GitHub 上
   GitHub 是一個倉庫的概念，可以想像成是上面說的你和阿嘎的共用資料夾，這就是你們兩個的共用倉庫。要將你電腦的東西放到網路上的倉庫中叫做 push。如果你沒有設定其他分支，這個指令會把你電腦中預設的 master 分支底下的檔案 push 到  GitHub 上，如果你有設定其他分支並把檔案 commit 在那個分支底下的話，就 `git push origin 分支名稱`，就可以了。
7. git pull origin master：把有人在 GitHub 上的更新拉下來本地電腦。
   把網路上倉庫的東西拉進來你的電腦中叫做 pu11。如果阿嘎也在你們的共用倉庫中 push 了他的檔案，那你可以用 pull 把他的檔案拉進來你的電腦中。一樣如果你們
   有設其他分支的話，就 `git pull origin 分支名稱`，就可以了。
